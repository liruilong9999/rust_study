# 学习笔记整理说明

## 1. 文档与注释要求

- 所有文档必须使用中文编写。
- 专业术语第一次出现时，需要写成“英文（中文说明）”格式。
- 代码需要补充中文注释，包括文件注释、函数注释、关键逻辑注释，以及必要的成员说明。
- 代码中的打印信息统一使用中文。
- Git 提交信息统一使用中文。

## 2. Rust 学习笔记目录约定

当前内容按学习阶段组织：

- `code/a00_install`：环境安装、工具链、包管理、Cargo 命令
- `code/a01_helloworld`：第 1 阶段，基础程序入口与输出
- `code/a02_guessing_name`：第 2 阶段，输入、循环、匹配、类型转换

后续新增阶段时，请继续沿用 `aXX_主题名` 的目录命名方式。

## 3. 每个阶段必须包含的内容

每个阶段目录至少包含以下内容：

1. 可运行的示例代码
2. 一个中文学习文档，说明本阶段学习目标、核心语法、运行方式
3. 如有依赖，必须在文档中说明依赖用途和常见命令。
4. 如果某一章使用了第三方库，必须在该章文档中补充对应的依赖安装命令，格式优先使用 `cargo add xxx`，如需固定版本可以写成 `cargo add xxx@版本号`。

建议阶段文档文件名统一使用“目录名_学习笔记.md”格式。

## 4. README 导航维护要求

根目录 `README.md` 必须承担学习导航作用，至少包括：

- 笔记简介
- 学习路径总览
- 每个阶段的入口链接
- 建议学习顺序
- 每个阶段的核心语法摘要

新增阶段后，必须同步更新 `README.md`。
